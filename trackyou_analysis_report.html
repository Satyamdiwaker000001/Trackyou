Critical bugs to fix now
Bugs & security issues
Hardcoded JWT secret in source code
auth.js fallback 'trackyou_secret_key_123' is committed to git. Anyone who forks your repo can forge tokens. Make JWT_SECRET required — crash on startup if the env var is missing, never use a fallback.
Task update uses req.id instead of req.params.id
In routes/tasks.js PUT handler: Task.findOne({ _id: req.id || req.params.id ... }). req.id is always undefined. The || masks the bug but the intent is clearly req.params.id only. Silently broken if req.params.id is absent.
CORS is wide open — allow all origins
app.use(cors()) with no config allows requests from any domain. Lock it down: cors({ origin: process.env.ALLOWED_ORIGIN }) in production so only your frontend can call the API.
"Focus Time" metric is fake
Dashboard shows (completedTasks × 1.5).toFixed(1)h as "Productive hours logged." This is entirely fabricated — there's no actual time tracking. Either implement a real timer or remove the metric. Misleading users erodes trust quickly.
Analytics chart uses createdAt, not completedAt
getDynamicChartData() checks t.updatedAt || t.createdAt to count daily completions. But the Task model has no updatedAt field — Mongoose doesn't auto-add it unless you set timestamps: true. All completed-task counts land on creation day, not completion day.
Current feature improvements
Existing features to improve
Add priority & tags to tasks
Tasks currently only have title, description, deadline, and completed. Add priority: enum['low','medium','high','urgent'] and tags: [String] to the Task model. Show urgency visually on TaskCard with a colored left border. Filter bar gains priority dropdown and tag chips.
Link tasks to projects
Projects and tasks are completely disconnected right now. Add projectId: ObjectId (optional) to the Task model. The task creation modal gets a "Project" dropdown. Project cards on the Projects page should show task count + completion ring, not just title and description.
Add inline task editing
TaskCard has no edit button — users can only toggle complete or delete. Add an edit flow: clicking a pencil icon opens the same modal pre-filled with task data. Reuse updateTask() which already supports title/description/deadline changes.
Calendar / timeline view for tasks
Right now all tasks are in a flat grid. Add a calendar toggle on the Tasks page using a library like react-big-calendar or a custom 7-day strip. Tasks positioned by deadline date give a much better sense of upcoming load at a glance.
Expand analytics with real data
The Analytics tab is just the same area chart repeated at a bigger size. Add: completion rate trend over 30 days (line chart), tasks by priority breakdown (bar chart), average time-to-complete (requires completedAt field), and streak counter (consecutive days with ≥1 completed task).
Implement "Change Password" (it currently shows alert)
Settings page has the button but calls alert("coming soon"). Add a POST /api/auth/change-password endpoint — verify current password, hash new one, save. Frontend modal with current password + new password + confirm fields. Also implement the "Forgot password" link on the login page.
New features to build
Suggested new features
Recurring tasks (daily / weekly / monthly)
Core to a daily habit tracker. Add recurrence: { type: 'daily'|'weekly'|'monthly'|'none', endDate } to Task. Cron job auto-creates the next instance when one is marked complete. Essential for the "daily routine" use case you already have in the UI copy.
Streak & habit tracking
Users log or link tasks to habits (e.g. "morning run", "reading"). Backend tracks currentStreak and longestStreak on a Habit model. Overview page shows a GitHub-style heatmap of daily completions and a flame icon with streak count — highly engaging for retention.
Pomodoro / focus timer
Add a 25/5 minute timer to each task. Timer state lives in React, plays a sound on completion, auto-logs a session to a FocusSessions collection. This replaces the fake "Focus Time" metric with real data and makes the app sticky daily.
Daily journal / end-of-day log
A simple text note per day (max 500 chars). Backend: JournalEntry model with userId, date, content. UI: a textarea in the Overview page footer section. Viewable in a calendar strip on the Analytics tab. Adds depth to the "track your day" value proposition.
In-app notification center
Right now all notifications go only to email. Add a bell icon in the Header with a badge count. Clicking opens a slide-over panel showing: upcoming deadlines, overdue tasks, streak milestones. Store notifications in MongoDB, mark as read. Can be implemented before WebSockets using polling.
Data export (CSV / PDF)
Add GET /api/tasks/export?format=csv — streams a CSV of the user's task history. PDF using pdfkit for a formatted weekly report. Frontend Settings page gets an "Export my data" section. Simple to implement, valuable for power users and a GDPR win.
PWA support
Add a Vite PWA plugin (vite-plugin-pwa), a manifest.json, and a service worker with offline caching. Users can "install" TrackYourDay on their phone home screen. Push notifications via the Web Push API can replace the cron email approach for deadline alerts.
Sub-tasks (checklist inside a task)
Add subtasks: [{ title: String, completed: Boolean }] to the Task model. TaskCard renders a mini checklist. The parent task's circular progress ring reflects subtask completion %, giving a richer sense of progress on multi-step work.
Design & UX recommendations
UI / design upgrades
TaskCard urgency visual system
Currently all TaskCards look identical regardless of state. Add a left-border color: red = overdue, amber = due within 24h, green = completed. Add a subtle pulsing animation on overdue cards. This alone dramatically improves scan-ability without a redesign.
Kanban view for tasks (not just grid)
A 3-column Kanban board (To-do → In Progress → Done) using @dnd-kit/core for drag-and-drop gives a visual workflow that flat task grids can't. Toggle between grid and kanban views with a button pair in the Tasks header.
Persist dark mode preference
The dark mode toggle in Settings applies via document.body.classList but doesn't persist — refreshing resets to dark. Save preference to localStorage and read it on app init in App.jsx. Also respect prefers-color-scheme media query for first-time users.
Skeleton loaders instead of empty states on mount
Dashboard fetches profile + tasks + projects in parallel on mount — during that window the UI shows nothing. Add skeleton shimmer cards where metric cards and task cards will appear. Much more polished than a flash of empty content.
Replace all window.alert / window.confirm with custom modals
The app uses browser alert() for errors and confirm() for delete confirmations throughout. These look completely out of place with the polished dark glass UI. A simple Toast component for errors and a styled ConfirmDialog component will massively improve perceived quality.
Responsive / mobile layout
The sidebar + main panel layout will likely break on mobile — no media queries are visible in the code review. Add a hamburger menu that collapses the sidebar to an overlay on screens under 768px. The task grid should go to single-column. Critical if you plan PWA support.
